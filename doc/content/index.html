Spekulatio

An almost totally unopinionated static site generator

Spekulatio is a static site generator that gives you all the freedom. Create your HTML without having to follow any rules imposed by the tool. Design the file structure of your project to match your content. Spekulatio doesn't require a predefined set of directories such as assets, static or posts, for example. Also, there's no need to fit your content to a fixed content taxonomy: you don't have posts, sections or subsections? That's fine, the tool won't force your content into them... or any.

Quickstart

To create the smallest site possible with Spekulatio, just create a directory where to place your content (content/) and a directory where the result site will be generated (build/). Add also some content in Markdown format. For example: 

    hello-world/
        build/
        content/
            index.md

Now, just execute spekulatio build inside hello-world/ and your brand new site will be ready: 

    hello-world/
        build/
            index.html
        content/
            index.md

To serve your site, execute spekulatio serve from the root directory. A HTTP server will be started and you'll be able to reach your converted files using any browser:

http://localhost:8000/index.html

Ok, ok, at this point your site may not look very impressive, but from here you can use templates to provide your HTML, add static files, define metadata or configure how each filetype should be processed. One advantage of Spekulatio is that simple things are easy to do and complex things are, well, not difficult either.

How it works

The main idea behind Spekulatio is very simple. Provide a directory with your contents —eg. Markdown, RestructuredText, HTML, CSS or JS files— and Spekulatio will go over each file converting them according to their file type —eg. converting Markdown to HTML or compiling SCSS to CSS.

The file structure of the input directories is up to you, Spekulatio will just generate a site that mirrors that structure using the converted files: 

    content/
        static/
            styles.scss
        one-section/
            page.md
        index.md
    
    build/
        static/
            styles.css
        one-section/
            page.html
        index.html

On top of this, Spekulatio provides several concepts that you can combine to create more complex sites: templates, values, input directories and actions.

Templates

Spekulatio supports Markdown, RestructuredText, JSON, YAML and HTML formats to create the HTML pages of your site. For that, it uses the awesome Jinja template engine. Set which template you want to use for a single content file of a whole section of your site and Spekulatio will render the final page using it.

Values

Spekulatio treats your site as a tree of nodes and you can set variables inside any node. For example, you can define author: me in one page and author: you in another. Which variables to define is up to you and, once defined, you can use them in templates to change how the final page is rendered or to display their information.

More importantly, these values are inherited. For example, if you set section: one in one folder, each page that is a descendant to it will have access to that value. Or, for instance, if you set site_name: foobar at the root of your project, every single page of the site will receive this value.

Input directories

You can pass multiple input directories to create a single site and Spekulatio will merge them together. This allows you to have one or more input directories that you can reuse across sites. Do you have a HTML layout and a set of CSS files that you can reuse in multiple projects? A theme in Spekulatio is not different from any other input directory.

Actions

You can configure which action should be performed for each file type in each input directory. For example, you can treat HTML files as templates in one directory and as content in another. JSON files can be used to generate HTML pages or just being directly copied into the output site. There are a number of actions available in Spekulatio, like copy, ignore, use as template, or render and you can decide how they are applied to the different file types.

Spekulatio's Theme

If you just want to create a site using some content but don't want to define all the HTML/CSS from scratch, Spekulatio also provides a generic *theme* that you can use out of the box. It's clean layout and customization posibilities make it a good choice for those times when you need to create a site fast. It is responsive and it automatically adapts well to desktop and mobile screens.

If you want to see how it looks like, just look at this site. It uses it!

Who is Spekulatio for?

There are, quite literally, hundreds of static site generators out there. Choosing among them can be really challenging. Spekulato can be a good fit if:

  • You don't want to fight against the tool to be able to create the exact HTML/CSS that you want or to define the file structure that better fits your project.
  • You favor simplicity and small tools with a reduced set of combinable concepts over hundreds of configuration options.
On the other hand, there are tools that may be a better choice if:
  • You prefer bigger applications with a large plugin ecosystem.
  • You're looking for a JavaScript based tool. There are a large number of static site generators that leverage the Node.js/React.js ecosystem. Spekulatio is not one of them though (you can add as much JavaScript as you want to your sites in Spekulatio, but the tool itself is language agnostic).
