Spekulatio

The you-cook-it-you-eat-it static site generator!
Spekulatio is a static site generator that gives you the freedom to arrange your input files in any way you want, write your HTML without having to follow any predefined structure and create your own configuration variables instead of being given a long list of them.

Description

Spekulatio uses Markdown, reStructuredText, JSON, YAML or plain HTML as content files, Jinja as its template engine and supports SASS/SCSS to generate the styles of the site.

There are plenty of static site generators out there. What makes Spekulatio different is that it doesn't give you instructions on how to build your site. Instead, it gives you a set of concepts that you can combine to tell it how to build it: input directories, actions, templates and values.

Input Directories: Choose Your Own File Structure

Spekulatio creates the output site by transforming the files provided in input directories but keeping their same file structure:

Actions: Convert Filetypes According to Your Needs

You can provide multiple input directories. And you can assign different sets of actions (ie. transformations) to each one of them. For example, HTML files can be copied without transformation in one directory, used as content in another and as templates in a third one:

Templates: Write HTML Any Way You Want

There's no pre-existing structure that you need to follow when writing your HTML. Write your templates using the Jinja template engine without having to adapt it to how the tool expects it to be.

Access the _node object to have full access to the site tree in memory and create links and menus automatically depending on the content.

{% filter escape -%} {% raw -%} {{ _node.content }} {%- endraw %} {%- endfilter %} layout.html

Values: Create Your Own Configuration Parameters

Spekulatio doesn't have too many configuration variables. Instead, it allows you to define your own values and change how the site is generated by your own.

For example, you can define an author variable or a show_banner one at any point, and the variables are then recursively inherited by any descendant page or directory from that point on. You can also overwrite previously defined variables and alter their value. Then, in templates, you have if conditionals, for loops and other constructs to both display the values of those variables or to change how the site is generated.

Is Spekulatio for me?

Rumor says that there are as many static site generators as sites themselves in Internet. So, in case you're trying to figure out if Spekulatio is a good fit for you, here you have a quick check list:

  • Do you want a static site generator with an abundance of themes and extensions?
    Then Spekulatio may not be what you're looking for. There are very mature solutions out there with big ecosystems. Spekulatio is more about creating your site from the ground up. It gives you a lot of control at the expense of giving you pre-cooked solutions. If you ever found yourself fighting with other tools, spending more time trying to adapt them than creating your site the way you want, then Spekulatio could be a good option though.
  • Do you want a static site generator that leverages the JavaScript ecosystem?
    There are many popular static site generators created around JavaScript tooling such as Node.js, React or Vue. In those cases you typically use JavaScript itself to configure the tool and create the visual components. In Spekulatio, you can, of course, add JavaScript to your sites, but it is designed for you to write good old HTML and CSS directly and provide the content in a variety of formats (Markdown, reStructuredText, YAML, JSON or HTML itself).
  • Do you like small, unopinionated tools?
    Then Spekulatio may be for you. Its intended design is to be minimalistic and it tries to make as few assumptions about your site as possible. You can, pretty much, write your site from scratch: the HTML that you want, with the file structure that you want and the parameters that you decide to use. It doesn't give you much more than that tough. Under the hood, Spekulatio just transform files from one format to another, giving you the option to pass values around to parametrize the whole operation. The creativity part is on you.

Do I have to create all my site from scratch?

Not really. If you want to create something fast, drop content files and generate a site out of them, then you can use the built-in Spekulatio theme. When using it, you don't have to write your own HTML and it gives you:

  • A clean visual appearance.
  • Responsive/Mobile layout out of the box.
  • Autogenerated menus: just drop your content files and the site navigation will adapt automatically.

If you want to know how it looks like, well... you're looking at it now. The entire Spekulatio documentation is generated using it.
